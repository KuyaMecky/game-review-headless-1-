# Contributing

Thank you for considering contributing to this project! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Install dependencies:
   ```bash
   npm install
   ```
5. Make your changes and test locally with `npm run dev`

## Development Guidelines

- Follow the existing code style and patterns
- Use TypeScript for all new files
- Use Tailwind CSS utility classes for styling
- Keep components modular and reusable
- Ensure all pages are responsive (mobile-first)

## Commit Messages

Use clear, descriptive commit messages:

```
feat: add casino comparison filter
fix: resolve mobile navigation overlap
docs: update deployment instructions
```

## Pull Requests

1. Ensure your code builds without errors (`npm run build`)
2. Update documentation if you change any setup steps
3. Describe your changes clearly in the PR description
4. Reference any related issues

## Reporting Issues

- Use GitHub Issues to report bugs
- Include steps to reproduce the issue
- Include browser/OS information if relevant
- Attach screenshots for UI issues
